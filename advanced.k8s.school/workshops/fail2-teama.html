How to use these files

  • Download: curl -f URL >filename
    • -f: ensures you are not downloading an error message
    • chmod 600 filename: needed to make ssh happy
  • SSH into a VM: ssh -F /dev/null -o UserKnownHostsFile=/dev/null -o StrictHostKeyChecking=no -o IdentitiesOnly=yes -i failX-Y.ssh trainee@IP
    • -F /dev/null: skips loading your default config file
    • -o UserKnownHostsFile=/dev/null -o StrictHostKeyChecking=no: skips host key checking
    • -o IdentitiesOnly=yes: skips trying keys other than the one specified on the command line
  • Connect with kubectl: kubectl --kubeconfig=failX-Y.conf ...
    • Optionally, you can use: export KUBECONFIG=failX-Y.conf
    • On the master VM, the trainee account is already configured with admin credentials (/opt/admin.conf); the root is not
